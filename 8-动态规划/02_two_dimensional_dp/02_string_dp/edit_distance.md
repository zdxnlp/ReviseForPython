# 编辑距离

## 题目

给定两个单词 `word1` 和 `word2`，求把 `word1` 转换成 `word2` 所需的最少操作数。允许操作为插入、删除、替换。

## 为什么是二维 DP

问题依赖两个字符串前缀之间的最优编辑成本，因此状态需要两个维度。

## 四步法

1. 定义状态：`dp[i][j]` 表示 `word1` 前 `i` 个字符转换成 `word2` 前 `j` 个字符的最少操作数。
2. 写转移：
   - 若末尾字符相等，`dp[i][j] = dp[i - 1][j - 1]`
   - 否则在插入、删除、替换三种操作中取最小值：`dp[i][j] = min(dp[i][j - 1], dp[i - 1][j], dp[i - 1][j - 1]) + 1`
3. 初始化：`dp[i][0] = i`，`dp[0][j] = j`。
4. 遍历顺序：按前缀长度递推，答案是 `dp[m][n]`。

## 递推阶梯

- 删除：把 `word1` 的最后一个字符删掉
- 插入：向 `word1` 末尾插入一个和 `word2[j - 1]` 相同的字符
- 替换：把 `word1[i - 1]` 改成 `word2[j - 1]`

## 公式

- 相等时：`dp[i][j] = dp[i - 1][j - 1]`
- 不等时：`dp[i][j] = min(dp[i][j - 1], dp[i - 1][j], dp[i - 1][j - 1]) + 1`

## 复杂度

- 时间复杂度：`O(mn)`
- 空间复杂度：`O(mn)`
